Axure Prototype / PRD Workspace

客户需求到 PRD 工作台原型

目标是让产品经理从客户反馈出发，完成需求评分、用户故事、PRD 摘要和验收标准维护。原型采用左侧输入、右侧文档、底部输出物的布局。

核心用户产品经理 / 客户成功 / 测试
关键动作录入反馈 -> 评分 -> 生成 PRD
交互重点Tab 切换 + 状态回写

交互说明

  • 新增反馈：提交后刷新反馈池、P0 数量、用户故事和 PRD 完整度。
  • 需求评分：优先级 = 业务价值 + 紧急程度 - 实现复杂度。
  • Tab 切换：不跳页，仅切换 PRD 摘要区域内容。
  • 验收勾选：勾选后调用 PATCH 接口，状态回写并刷新报告。
