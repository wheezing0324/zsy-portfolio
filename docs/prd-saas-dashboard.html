PRD / SaaS Dashboard

SaaS 运营数据看板原型

面向 B 端 SaaS 运营和客户成功团队，用指标、分层、异常提醒和跟进任务支撑续费决策。

2 APIs 覆盖分层数据加载和客户跟进任务写入
背景 目标 用户 功能 Axure 接口 验收 测试部署

1. 产品背景

B 端 SaaS 运营和客户成功团队需要在续费前识别风险客户、查看使用活跃度并创建跟进动作。本作品展示数据产品方向的指标口径、筛选联动、客户分层和后端任务写入。

2. 产品目标

  • 用轻量看板呈现客户活跃、续费、风险和响应效率。
  • 通过客户分层筛选快速定位不同客户群。
  • 支持点击客户动作创建后端跟进记录。

3. 目标用户与核心场景

运营查看整体客户活跃和风险情况；客户成功识别需续费访谈或升级处理的客户；产品经理根据数据表现判断下一步迭代方向。

4. 功能需求

客户分层筛选

  • 提供全部客户、企业客户、风险客户三个筛选项。
  • 切换后刷新指标、趋势、客户列表和产品判断。

指标看板

  • 展示活跃账号、本月续费率、异常客户、平均响应时长。
  • 指标随客户分层变化。

客户列表

  • 展示客户名称、当前状态、建议动作。
  • 点击建议动作后创建后端跟进记录。

5. Axure 原型设计

01筛选器

全部、企业、风险客户切换。

02指标卡

活跃、续费、风险、响应效率。

03趋势图

近 5 日活跃变化。

04客户列表

客户、状态、动作。

05跟进任务

点击动作生成记录。

6. 数据与接口

GET/api/dashboard?segment=all

按客户分层返回指标、趋势、客户列表和产品判断。

POST/api/dashboard/followup

提交 segment、customer、action，创建客户跟进记录。

7. 验收标准

  • 点击不同客户分层后，四个指标卡同步变化。
  • 趋势图、客户列表和产品判断与当前分层一致。
  • 点击客户动作后，页面新增后端跟进记录。
  • 接口失败时页面仍可展示 fallback 数据。
  • 移动端下指标卡、趋势图、客户列表纵向排列。

8. 测试与部署

本地 Node 服务端口为 4173。主要测试覆盖筛选 GET、跟进 POST、列表刷新和移动端布局。部署方式为本地静态页面 + Node HTTP API。
